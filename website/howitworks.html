Software


1.App

  • Goal

    • Our system's Android app is desgined for users to control the fueling robot in the gas station. When users drive to gas station, they could activate the gas station assistant robot to fuel up their car from inside of their vehicle using the app.

  • UI Design

    • The layout of the app consists of six fragments: Main Page, Setting Page, Warning Page, Fueling Page, Complete Page and Assistance Page.

                  Fueling Page            Complete Page           Assistance Page

  • Usage

    1. When users open the App for the first time, they stay on the setting page and need to create a new user to access main page for further operations.
    2. Users input their personal information (a relevant errormessage pops up if any input is in incorrect format) and click on Add User button, a new user will be created.
    3. If users want to change information details of any created users, they can click on that specific user button first and then re-input new information, followed by clicking on Update User button. Modified user information will be shown on account summary.
    4. Users click on Return button to access main page.
    5. Users click on one of Fuel Type buttons and select fuel amount using either “Full” checkbox or amount slider (depends on how much gas they want), then click on Start Fuelling button to start. A prompt window will pop up for them to confirm everything is ready.
    6. After users confirm fueling, they stay on fueling page and could see a dynamic change of fuel amount and price. During this process, they can click on Stop button any time to stop fueling.
    7. When fueling is done, users stay on complete page and could click on Finish button to return main page. Optionally, they could select the checkbox to receive an email receipt.
    8. If users have any problems during fueling, click on Assistance button to ask for human help.
